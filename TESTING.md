# 快速测试指南

## 重要提示

**hybrid模式已修复！** 现在可以直接使用。

## 测试改进效果

### 1. 安装依赖（首次使用hybrid模式）

```bash
.\venv310\Scripts\Activate.ps1
pip install torchcrepe
```

**注意**: 如果未安装torchcrepe，系统会自动回退到纯RMVPE模式，不会报错。

### 2. 运行系统

```bash
python run.py
```

系统会自动使用新的hybrid F0方法。

### 3. 测试对比

建议使用之前出现问题的歌曲进行对比测试：

**测试重点**：
- 回声较重的段落是否还有杂音
- 固定段落是否还有丢音/撕裂
- 整体是否更自然，减少AI机械感

### 4. 参数微调（可选）

如果默认参数效果不理想，可以在UI中调整：

#### 场景A: 回声很重
```
F0方法: hybrid
保护强度: 0.25
索引率: 0.25
滤波半径: 1
```

#### 场景B: 高音技巧多
```
F0方法: hybrid
保护强度: 0.30
索引率: 0.30
滤波半径: 0
```

#### 场景C: 说唱/快节奏
```
F0方法: hybrid
保护强度: 0.35
索引率: 0.25
滤波半径: 0
```

### 5. 性能说明

- **显存**: hybrid模式额外占用约500MB
- **速度**: 比纯RMVPE慢15-20%
- **质量**: 明显提升，特别是回声段落

### 6. 回退方案

如果遇到问题，可以临时回退到旧配置：

在UI中设置：
- F0方法: rmvpe
- 滤波半径: 3
- 保护强度: 0.45
- 索引率: 0.35

## 预期改进

1. **回声段落杂音**: 应该明显减少或消失
2. **丢音/撕裂**: 高能量段落保留更多细节
3. **自然度**: 保留颤音，减少机械感
4. **辅音清晰度**: 更好的辅音保护

1. 问题出现的具体时间点（如：1:23-1:28）
2. 使用的参数配置
3. 原主唱该段落的特征（如：高音、快速、回声重等）
